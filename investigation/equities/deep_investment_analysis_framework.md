# QuantPyLab 深度投资价值分析框架 (SOP v5 - Institutional Standard)

本框架旨在指导 AI 模拟买方机构级研究流程，产出具备严肃定价权的长篇深度研报。

## 工作规则

1. 分析工作开始之前，先使用`uv run main.py sync-all --symbol 股票代码` 同步数据，股票代码是6位纯数字。检查数据同步的输出，如果有任何报错则最多重试三次，如果始终无法完全成功则停止工作，报告用户。
2. **异步迭代产出机制**：为了突破单次输出篇幅限制并确保研究深度，严禁在所有调研结束后一次性合成。必须在每个核心模块研究结束后，立即输出对应的长篇章节文档（要求数据密度极高，单章字数 1500+），作为最终报告的基石。在每个模块结束后重新读取本文档强化规则。

## 模块二：时间维度——产业与库存周期 (Time: Cycle Analysis)

**目标**：识别公司所在的行业与上下游。推演目标年份产业与下游情形变化。

1. **产业周期**：行业生命周期定位；技术迭代路径风险。
2. **库存周期**：产业链库存水平（去库 vs 补库）；周转天数边际变化。
3. **下游景气度**：必须考虑公司的产品和服务对应的下游产品和终端需求的景气度，下游销量和价格是增长还是减少？。

**【执行建议】**: 灵活构造关键词使用 `google_web_search` 搜索相关信息。

## 模块三：空间维度——竞争格局与地位 (Space: Competitive Landscape)

**目标**：评估公司在博弈场中的生存能力与获利空间。

1. **竞争格局扫描**：集中度变动 (CR3/CR5)；竞品的市占率、技术水平及资本开支情况。
2. **护城护识别**：壁垒是基于成本规模、品牌溢价、转换成本，还是专利/特许经营权。
3. **产业链定价权**：对上游压款能力及对下游的议价能力（通过财报应付/预收科目验证）。

可以下载竞争对手公司的财报以做对比分析

**【外部数据交叉验证启发式 (Cross-Verification Heuristics)】**:
利用 `google_web_search` 捕获非财务信息时，应从以下维度动态构造关键词组合，严禁仅依赖财报描述：

* **真实订单验证**：寻找在手订单（Backlog）的实质证据，关注行业峰会、招标公示及核心客户的采购季谈话。

* **实物产能校验**：搜索土地招拍挂、重大工程环评公示、新增变压器/动力的能源消耗信号，验证扩产逻辑的物理进度。

**【输出指令】**：在模块三结束时，立即撰写并保存为文档 **`investigation/equities/<公司名称>/reports/研报章节二：产业周期与竞争格局.md`**。要求：包含详尽的竞品参数对比表，基于非财务信息（如招标网、供应链传闻）验证市场份额。

## 模块五：增长驱动与盈利预测 (Growth Drivers & Financial Modeling)

**目标**：构建基于全维调研证据链的财务模型，实现定性洞察向定量定价的严密跨越。给出目标年份的量、价、成本、利润等数据。

1. **分业务 P*Q 模型的证据链锚定**：
   * **销量 (Q) 确定性溯源**：严禁孤立看待增速。预测需综合多维证据：1) **内生动力**（产能利用率上限、产线建设节奏、在建工程转固信号）；2) **外生拉动**（下游行业资本开支周期、政府产业补贴与专项政策、大客户份额锁定意向）；3) **博弈空间**（行业集中度变化、竞品供给缺口或出清速度）。
   * **价格 (P) 趋势逻辑推演**：通过调研还原定价权的本质。考量：1) **成本传导**（原材料波动对毛利率的边际影响与滞后时长）；2) **竞争烈度**（低价搅局者的生存状态、行业协同定价机制的有效性）；3) **价值升级**（产品结构从“量”向“质”的切换、高附加值新品对 ASP 的拉动效应）。
2. **利润矩阵的边际穿透**：
   * **盈利质量研判**：通过调研识别毛利率波动的真因（是规模效应、技术降本，还是短期政策红利）。
   * **费用结构的弹韧性**：评估研发投入与未来营收增长的边际产出比，预判规模效应下的费率下行曲线。
3. **多态逻辑情景推演 (Dynamic Scenario Planning)**：
   * **证据组合测试**：基于调研中发现的潜在变量（如：地缘政策突变、核心设备交付延期、新赛道渗透率超预期）构建 **保守 (Bear) / 中性 (Base) / 乐观 (Bull)** 利润矩阵。
4. **EPS 预测与逻辑索引**：给出目标年份的 EPS 预测值，并为每一项核心假设标注调研证据的来源编号（证据回溯制），确保逻辑链条可复核、可证伪。
5. **分部估值 (SOTP) 利润分配算法规范**：
   当且仅当公司具备多个不同属性的业务板块时，使用以下算法模拟分部净利润，严禁主观盲目拆分：
   * **计算公式**：
     `板块模拟净利_i = 板块营收_i * [板块毛利率_i - (总期间费用率 * K_i)] * (1 - 有效税率)`
   * **系数分配原则 (K_i)**：
     * `K = 1.0`：适用于成熟、标准化的制造业务。
     * `K = 1.2-1.5`：适用于高研发强度、高管理难度的成长期/孵化期业务（如数控机床）。
     * `K = 0.5-0.8`：适用于具备极强规模效应、费用极低的原材料或单一产品批发业务。
   * **勾稽校验**：`∑(板块模拟净利_i) ≈ 公司预测总净利润`。
6. **证据链强度与确定性评分 (Certainty Scoring)**：
   * 对每一个核心增长驱动力（如：产能扩张、新产品放量、行业价格回升）进行确定性评分（1-5分）。
   * **5分（极高）**：有已落地的重大合同、在建工程已转固、核心客户定点函、或已公告的排产数据。
   * **3分（中等）**：行业趋势确定且具备竞争壁垒，但尚未体现为确定的在手订单。
   * **1分（极低）**：高度依赖外部宏观变量或管理层愿景，缺乏底层实物资产或合同支撑。

**【输出指令】**：在模块五结束时，立即撰写并保存为文档 **`investigation/equities/<公司名称>/reports/研报章节四：增长驱动与盈利预测.md`**。要求：展示详尽的 P*Q 模型假设表，包含每一核心假设的确定性评分与分部利润模拟过程，并标注证据来源。

## 模块六：估值定价与全变量风险对冲 (Valuation & Hedge Audit)

**目标**：基于业绩预测、时间窗口规则及适配性对冲矩阵，给出严谨的定价建议。

**1. 价值锚定：适配性方法选择 (Adaptive Anchoring)**：
* **方法论工具箱**：AI 必须根据公司的**商业模式、资产属性、现金流特征及所处生命周期**，自主选择最匹配的估值路径（建议组合使用 2 种或以上进行交叉验证）：
    * **可选工具**：PE (市盈率)、PB-ROE (市净率-ROE 盈利模型)、DCF (现金流折现)、SOTP (分部估值)、PEG (成长性估值)、PS (市销率)、EV/EBITDA 等。
    * **必选校验：历史分位审计**。必须调取该股近 5-10 年的估值分位（PE/PB 等），识别当前估值在长周期中的坐标。
* **估值锚点切换 (Rolling Strategy)**：
    * **当年 (Q1-Q3)**：锚定当年 (Current Year) EPS/BVPS。
    * **切换期 (Q4)**：自动滚算至次年 (Next Year) 数据。
* **执行指令**：简述你选择这些估值方法的理由，并给出“基准市值”区间。

**2. 核心：估值全变量对冲审计矩阵 (Comprehensive Audit Matrix) —— 强制性输出**：
* **审计原则**：将基准市值作为“底色”，通过对以下维度进行定性转量化的审计，计算出最终的**修正目标市值**。
* **估值影响力过滤 (Valuation Influence Filtering)**：在执行调节前，必须进行**审慎思考**。发掘出的利好或利空因子绝大多数只是市场杂音，并不能改变公司的估值中枢。真正能够享受到估值溢价或折价的因子是极少数，大部分因子对估值的影响其实为零。要时刻防止将“平庸的关联”误认为“定价的锚点”。
* **审计公式**：修正目标市值 = 基准市值 * (1 + Σ调节幅度)。

| 审计维度 | 核心审计因子 (示例，AI 需根据实况调整) | **审慎思考 (估值影响力过滤)** | 调节建议 | 深度理由 (PE/PB 贴现维度) |
| :--- | :--- | :--- | :---: | :--- |
| **内部：企业底层特质 (DNA)** | **AI 自行挖掘：** 如战略定力、内生孵化能力、资本配置效率、治理透明度、文化护城河等。 | **辨析：** 该特质是否真的具备改变估值中枢的能力？剔除那些“平庸的优秀”，识别真正能带来溢价的稀缺性。 | +/- % | 评估企业内在特质是增加了长期确定性（溢价），还是隐含了治理风险（折价）。 |
| **宏观：政策与战略对齐** | **AI 关联审计：** 行业受政策扶持或压制的程度、税收稳定性、国家战略导向关联度等。 | **辨析：** 政策关联是否真的能转化为定价权？绝大部分政策红利对具体公司的估值影响其实为零，判断其是否属于那能产生溢价的少数。 | +/- % | 评估政策环境对该行业整体“无风险利率”或“远期天花板”的影响。 |
| **宏观：地缘与外部约束** | **AI 风险对冲：** 出口/供应链依赖度、地缘摩擦敏感度、海外资产安全、国际转口审查等。 | **辨析：** 风险因子是否真的构成了长期性的折价，还是仅为短期情绪噪声？判断其对公司生存底线的实质影响。 | +/- % | 将“无法量化为具体成本”的宏观焦虑，显性化为估值倍数的风险扣除。 |
| **市场：流动性与风格** | **AI 审美对齐：** 机构持股集中度、自由流通盘规模、当前市场风格（红利/成长/价值）契合度。 | **辨析：** 所谓的“风格契合”是否只是短期资金博弈？判断其是否真的能带来可持续的溢价，而非随风而逝的泡沫。 | +/- % | 评估流动性溢价或风格错配导致的阶段性估值弹性。 |
| **综合调节系数** | —— | —— | **Σ%** | **对冲后的最终修正乘数。** |

**3. 强制性：逻辑纯净性校验 (Consistency Guardrails)**：
* **严禁重复计算**：AI 必须明确区分哪些利好/利空已在“利润预测（E）”中体现，哪些必须在“估值倍数（PE）”中体现。
    * *准则*：具体的、可计算的钱（增减）进 E；不可计算的、关乎信心与不确定性的因素进 PE。
* **锚点回测**：若修正后的隐含综合 PE 显著偏离历史中枢或同业水平，必须给出具备深厚商业洞察的逻辑支撑。

**4. 确定性与胜率矩阵 (Win Rate & Risk-Reward)**：
* **三态逻辑加权**：为主观赋予**保守/中性/乐观**场景的发生概率（∑P=100%）。
* **最终目标价合成**：基于修正市值给出目标价，计算盈亏比 `(目标价 - 现价) / (现价 - 保守场景价格)`。

**【输出指令】**：在模块六结束时，立即撰写并保存为文档 **`investigation/equities/<公司名称>/reports/研报章节五：估值定价分析.md`**。要求：清晰阐述适配方法、展示完整的全变量审计矩阵、记录目标价计算过程及关键假设。

## 模块七：技术面分析 (Technical Analysis)

**目标**：通过价格行为分析识别当前股价的趋势特征，预判未来的短期与长期走势。

* **核心逻辑维度**：
  1. **趋势架构分析**：研判当前处于“趋势延续”、“结构反转”还是“区间震荡”。识别当前级别（日线/周线）的主导趋势，并判断趋势的强弱状态。
  2. **关键支撑与阻力**：识别历史上具有强烈心理暗示与资金博弈性质的点位（如历史极值、成交密集区、长周期均线支撑）。
  3. **量价行为解读**：分析成交量与价格变动的配合关系，寻找资金流入/流出的蛛丝马迹，判断当前价格波动的动能是否充沛，是否存在量价背离信号。
  4. **未来走势演绎**：
     * **短期走势（1-3个月）**：基于当前动能与阻力位，推演可能的波动区间与节奏。
     * **长期走势（6-12个月以上）**：结合大级别形态，预判股价运行的潜在路径（如：圆弧底形成、上升通道维持等）。

**【执行建议】**: 灵活调用编写针对性统计代码，提取价格行为（Price Action）特征，严禁单纯依赖单一技术指标。

**【输出指令】**：在模块七结束时，立即撰写并保存为文档 **`investigation/equities/<公司名称>/reports/研报章节六：技术面分析.md`**。明确指出短期关键位与长期走势研判。

## 模块八：深度研报合成标准 (Long-form Report Synthesis)
